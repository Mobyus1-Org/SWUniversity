## Method & caveats

- A leader counts as **implemented** if its canonical card ID is referenced anywhere in the engine source (`src/server/engine/**`) or the Puzzles UI leader registry (`src/containers/PuzzlesPage.tsx`). No reference anywhere → listed below as unimplemented.
- **Canonical IDs only.** Promo / alt-art / reprint printings that alias a base leader via `card-db/overrides-generated.ts` (`P25_*`, `GG_T*`, `C24_*`, `C25_*`, …) are excluded — they inherit the canonical leader's status and are not separate work.
- This is a **reference scan, not a per-clause audit.** A leader marked implemented may still be only partially wired (front side without the deployed leader-unit side, or vice versa) — partial implementations are invisible. Spot-check a leader's actual coverage before assuming it's done, and confirm a listed one is truly unreferenced (not merely referenced incidentally) before starting.
- Each leader needs **both sides**: the front (undeployed) ability AND the deployed leader-unit ability.
